# AI Context

This document defines the minimal context required for AI agents to understand and work inside the UMKM OS Blueprint repository.

## Purpose

Provide a compact, high-signal overview of:
- Project goals
- Architectural direction
- Constraints
- Working principles

## Project Summary

UMKM OS is a modular operating system for MSMEs in Indonesia, designed for:
- Offline-first usage
- Ultra-fast POS operations
- Edge computing compatibility
- AI-assisted workflows

## Core Principles

- Merchant First
- Offline Always
- Speed > Features
- Simplicity Wins
- AI is an assistant, not an authority

## AI Behavior Rules

- Always read START_HERE.md first
- Prefer simplicity over optimization
- Never introduce unnecessary complexity
- All features must reduce merchant effort or time

## Context Scope

AI agents should assume:
- Unstable internet environments
- Low-end devices
- High transaction speed requirements
